# 🔑 Как подключить мерчанта вручную через Clover API token

> Способ обойти OAuth + production verification.
> Берёшь токен напрямую через Clover dashboard, вставляешь в наш дашборд, бот работает.

**Кому подходит:** проблемные мерчанты которые **постоянно** пишут (Plov Point, La Pollera, и т.п.) — даём боту полный доступ к их Clover, он сам разруливает большинство вопросов.

**Время:** ~5 минут на одного мерчанта (если у тебя есть admin-доступ к их Clover account).

---

## 🎯 Что нужно от мерчанта

**Один из вариантов:**

### Вариант А — у тебя уже есть admin-доступ
Ты как ISO часто можешь логиниться в их Clover dashboard через email типа `daler@infinitypay.us` или `support@infinitypay.us`. Если да — переходи к шагу 1.

### Вариант Б — попросить owner залогиниться вместе
Звонишь owner'у мерчанта (например Plov Point) и просишь:
> "Дайте 2 минуты — нужно настроить нашему боту доступ к вашей Clover, чтобы он автоматически отвечал на ваши вопросы. Зайдите на dashboard.clover.com своим логином."

→ дальше делаешь screen-share или говоришь по телефону что нажимать.

### Вариант В — owner делает сам
Шлёшь ему **этот мини-гайд** (см. `MERCHANT_INSTRUCTIONS.md` в той же папке) — он сам берёт токен и отправляет тебе в WhatsApp/Telegram.

## 📋 Шаг 1. Зайти в Clover dashboard мерчанта

URL: **https://www.clover.com**

1. Email + пароль владельца мерчанта (или твой ISO-доступ если такой есть)
2. После логина — выбрать конкретного мерчанта (если у owner несколько локаций)

---

## 📋 Шаг 2. Создать API Token

1. В Clover dashboard сверху-справа: **Account & Setup** (иконка человечка / шестерёнка)
2. В левом меню: **Setup → API Tokens**
   - Если не видишь — поищи "API" в search bar
3. Нажми **"Create New Token"**
4. Заполни форму:
   - **Name:** `Infinity Pay Bot Access` (или любое описательное имя)
   - **Permissions / Scopes** — отметь ВСЕ нужные галочки:
     - ✅ Read Merchant
     - ✅ Read Inventory
     - ✅ Write Inventory  ← важно для toggle/price changes
     - ✅ Read Payments
     - ✅ Read Orders
     - ✅ Write Orders (если есть)
     - ✅ Read Employees
     - ✅ Read Customers (опционально)

5. **Create Token** → откроется попап с длинной строкой типа:
   ```
   abc12345-def6-7890-ghij-klmnop123456
   ```

6. **СКОПИРУЙ ЕЁ СРАЗУ** — после закрытия попапа Clover её больше не покажет
7. Сохрани в надёжное место (1Password / временно в Notes)

---

## 📋 Шаг 3. Найти Clover Merchant ID

В том же Clover dashboard:

1. **Account & Setup → Merchant Info** (или просто `/merchant_id` в URL)
2. **Merchant ID:** строка из ~13 символов типа `5VHNT2Q0H8AJ1`
   - Это НЕ MID Tekcard (16-значный)
   - Это **внутренний Clover ID**

Также можно подсмотреть в URL:
```
https://www.clover.com/dashboard/merchant/5VHNT2Q0H8AJ1/...
                                            ^^^^^^^^^^^^^^^
```

Скопируй и сохрани.

---

## 📋 Шаг 4. Вставить в наш дашборд

1. Открой https://infinity-pay-dashboard-production.up.railway.app
2. Логин: `rozikov.shams@gmail.com`
3. **Merchants** → найди мерчанта (поиск по INF-XXXX или имени)
4. Клик на строку → откроется боковая панель
5. Вкладка **`Clover`**
6. Раздел **"Or paste token manually"**:
   - Поле 1: `Clover merchant ID` → вставляешь `5VHNT2Q0H8AJ1`
   - Поле 2: `Access token` → вставляешь `abc12345-def6-...`
7. **"Save & verify"**

### Что произойдёт

Дашборд **сразу** делает test-call в Clover (`GET /v3/merchants/{mid}` — забирает имя бизнеса). Если:

- ✅ **OK 200** — токен сохраняется зашифрованным, мерчант показывается как Connected (зелёная точка)
- ❌ **401 Unauthorized** — токен не работает (возможно опечатка)
- ❌ **403 Forbidden** — у токена нет нужных scopes (вернись и поставь все галочки)
- ❌ **404 Not Found** — Clover Merchant ID неверный

В случае ошибки — фиксишь и пробуешь снова. Дашборд **не сохранит** невалидный токен.

---

## 📋 Шаг 5. Проверить что бот видит данные

После save:

1. В том же дашборде, на той же вкладке `Clover`, **прокрути вниз** — должны появиться:
   - **Today's sales** (если за сегодня были транзакции)
   - **Yesterday's sales**
   - **30-day trend chart**
   - **Devices status** (какие терминалы онлайн)

2. Если данные грузятся — **бот тоже их видит**. Готово!

3. Тест с боком (опционально):
   - Скажи owner'у: "Напиши боту: какие продажи сегодня?"
   - Бот должен ответить с реальными цифрами

## ⚠️ Важно

### Безопасность
- Токен **зашифрован AES-256** в БД
- Никто кроме твоего дашборда его не видит (не Anthropic, не Telegram, не Railway админы)
- Если хочешь отозвать — пойди в Clover dashboard → API Tokens → Delete

### Лимиты Clover
- **1000 requests/min на app** — у нас далеко не упрёмся
- Если мерчант пишет боту 10 раз в час → ~30 API requests = OK

### Когда менять токен
- Если меняется владелец бизнеса → создай новый, удали старый
- Если кажется что utечка → создай новый, старый Clover автоматически invalidate-нет
- API tokens в Clover **не expire** автоматом (живут пока ты не удалишь)

### Ограничение
- ⚠️ Если у мерчанта **несколько локаций** — токен будет привязан к **одной**
- Для каждой локации нужен свой токен + своя запись в нашем дашборде

## 🎯 Шорткат — для пилотных мерчантов

Если ты хочешь подключить **3-5 пилотных мерчантов** прямо сегодня для теста:

```
1. Plov Point      (тестовый, у тебя есть доступ)
2. La Pollera      (тестовый, у тебя есть доступ)
3. [merchant 3]    — самый "заёбывающий"
4. [merchant 4]    — второй "заёбывающий"
5. [merchant 5]    — для разнообразия (другой тип бизнеса)
```

После подключения мониторь первую неделю — бот будет автоматически закрывать "сколько продал", "выключи блюдо", "когда депозит". Если 80%+ вопросов закрывается без эскалации — формат работает, можно подключать остальных.

## 🆘 Если что-то не получается

1. **"Token saved" но данных нет** → подожди 30 сек, перезагрузи страницу. Если всё равно пусто — у мерчанта нет транзакций за период.

2. **"Invalid Clover token"** → проверь что скопировал ВСЮ строку без пробелов. Пересоздай токен в Clover если сомневаешься.

3. **"Forbidden"** → у токена нет scope. Удали этот token, создай новый со всеми галочками.

4. **Owner не может найти "API Tokens" в меню** → у его tier-плана может не быть этой опции (Clover Lite иногда отключает). Тогда:
   - Попроси Clover support включить
   - ИЛИ используй standard OAuth (warning + Continue) — наш дашборд это тоже умеет

5. **Дашборд показывает "Save & verify" но кнопка серая** → проверь что у тебя role `owner` или `admin`. У `agent` роли нет прав на set-token.
